# QUICKSTART

Коротка інструкція для локального запуску фронтенд-додатку.

## Опис

Це Next.js (v14) застосунок, який працює з бекендом через OpenAPI-контракт та згенерований API-клієнт.

## Передумови

- Node.js >= 18
- npm (використовується `package-lock.json`)

## Встановлення

```bash
npm install
```

## Змінні середовища

Додаток використовує змінні середовища, які мають бути доступні під час запуску.

### Обовʼязкові

- `NEXT_PUBLIC_API_BASE_URL` — базовий URL бекенду (наприклад `http://localhost:3001`).
  - Має бути валідним URL.
  - Використовується API-клієнтом для всіх запитів.

### Приклад `.env.local`

```env
NEXT_PUBLIC_API_BASE_URL=http://localhost:3001
```

> Next.js автоматично підхоплює `.env.local` для локальної розробки.

## Запуск

### Режим розробки

```bash
npm run dev
```

За замовчуванням застосунок буде доступний на `http://localhost:3000`.

### Production-збірка

```bash
npm run build
npm run start
```

## OpenAPI та типи

Типи для API генеруються з OpenAPI-специфікації:

```bash
npm run api:generate
```

Запускай цю команду, якщо OpenAPI-файл `.vibe/openapi/openapi.json` було змінено.

## Лінтинг

```bash
npm run lint
```

## Типові проблеми

- **Помилка валідації env**: перевір, що `NEXT_PUBLIC_API_BASE_URL` задано і є валідним URL.
- **Помилки API**: переконайся, що бекенд запущений і доступний за вказаним base URL.
